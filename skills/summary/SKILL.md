# summary：链接内容提炼与知识库存档

抓取用户提供的链接，自动识别内容类型，提炼核心观点，写入本地知识库。

支持两种运行模式：
- **URL 模式**（用户触发）：抓取链接内容 → 提炼 → 写入
- **内联内容模式**（其他 skill 调用，如 learning）：内容已就绪，跳过抓取，直接写入

---

## 内联内容模式（由其他 Skill 调用时）

当另一个 skill 调用 summary 且已提供内容时：

1. **跳过 Step 0**（确认摘要）和 **Step 1**（抓取内容）
2. 直接进入 **Step 2**（识别类型）和 **Step 3**（写入文件）
3. 文件名规则：`~/knowledge/YYYY-MM-DD-{来源skill}-{主题}.md`（如 `learning-Rust`）
4. **调用方可覆盖信息卡字段**：若调用方提供了字段清单（包括字段顺序、值、需省略的字段），按调用方约定执行；调用方未指定的字段沿用 Step 3 默认规则
5. **调用方可指定 `类型` 为非 4 类的特殊值**（如 `学习计划`、`调研报告`），此时跳过 Step 2 的类型判断
6. 写入完成后告知用户文件路径

**触发判断**：调用方说明"内联模式"、"内容已就绪"、"无需抓取 URL"即视为内联模式。

## 执行流程（URL 模式）

### Step 0：信息收集

**先解析输入 → 再自补 → 确认后动手**

#### 解析清单

| 信息项 | 若缺失，如何处理 |
|-------|--------------|
| 链接 URL | **必须追问**，无法继续 |
| 阅读重点/关注角度 | 默认"全面提炼"，在摘要中标注 |
| 保存路径 | 默认 `~/knowledge/`，可在确认时修改 |

#### 理解摘要模板

解析完成后展示摘要，**紧跟 `AskUserQuestion` 确认**：

> 我的理解：
> - 链接：**[URL]**
> - 关注角度：**[全面提炼 / 用户指定角度]**
> - 保存路径：**`~/knowledge/YYYY-MM-DD-{标题}.md`**

```
AskUserQuestion({
  questions: [{
    header: "确认信息",
    question: "以上理解是否正确？",
    multiSelect: false,
    options: [
      { label: "确认，开始抓取", description: "信息无误，立即开始" },
      { label: "需要调整", description: "请在 Other 文本框中说明" }
    ]
  }]
})
```

<HARD-GATE>
未展示摘要并收到用户确认前，不得开始抓取。
</HARD-GATE>

### Step 1：抓取内容

按以下顺序尝试，失败后自动降级，无需询问用户：

#### 层级 1：chrome-mcp（优先）

用浏览器打开页面，利用已有登录态和 JS 渲染能力抓取：

```
mcp__chrome-mcp__get_page_content(url="{URL}")
```

若调用失败（connection refused / ECONNREFUSED / tool not found）：
1. `Bash("pkill -f mcp-chrome-bridge && sleep 2")`
2. 重试一次
3. 仍失败则跳至层级 2

判断失败的条件：返回内容为空、疑似登录页（含"请登录"/"sign in"/"403"等）、内容明显不完整（少于 200 字）。

#### 层级 2：WebFetch（chrome-mcp 失败时）

```
WebFetch({ url: "{URL}", prompt: "返回完整正文内容，保留文章结构" })
```

#### 层级 3：用户粘贴（两层均失败时）

停止抓取，告知用户：

> 自动抓取失败（可能是内容受限或需要特殊登录）。请将文章内容直接粘贴给我。

---

### Step 2：识别类型并提炼

#### 类型判断（按优先级匹配）

| 类型 | 判断信号 |
|------|---------|
| **技术教程** | 有步骤编号 / 代码块；标题含「如何 / 使用 / 入门 / 指南 / 教程」；以操作为主 |
| **原理架构** | 有设计讨论 / 系统对比 / 深度分析；标题含「原理 / 架构 / 设计 / 深入 / 分析」 |
| **行业资讯** | 事件驱动；含时间 / 数据 / 发布公告；主体是外部事件而非技术本身 |
| **思想观点** | 论点驱动；个人见解 / 方法论 / 反思；标题含「为什么 / 如何思考 / 我认为」 |

无法判断时默认「**原理架构**」。

#### 模板 A：技术教程

```markdown
## 🎯 功能定位

> 一句话说清这个工具/技术/框架做什么、解决什么问题。

## ⚡ 快速上手

核心操作步骤（3-5 步），关键代码片段保留：

1. **{步骤 1}** — {做什么 + 关键命令}
2. **{步骤 2}** — ...

## 🔧 核心 API / 命令

| API/命令 | 作用 | 关键参数 |
|---|---|---|
| ... | ... | ... |

## ⚠️ 常见坑

- **{坑 1}** — 现象 + 正确做法
- **{坑 2}** — ...

## 💡 适用场景

什么情况下应该用？哪些情况不适合？
```

#### 模板 B：原理架构

```markdown
## 🎯 核心设计理念

> 为什么要这样设计？解决了什么根本问题？

## 🏗️ 关键架构决策

3-5 个核心设计选择，每个写「是什么 + 为什么这样选」：

- **{决策 1}** — {内容}；选择理由：{...}
- **{决策 2}** — ...

## ⚖️ 技术权衡

**优势**

- ...

**劣势 / 局限**

- ...

## 🔄 与现有方案对比

| 维度 | 本方案 | 其他方案 |
|---|---|---|
| ... | ... | ... |

## 💡 启发与应用

这个设计理念能在自己的哪些场景中借鉴？
```

#### 模板 C：行业资讯

```markdown
## 📰 核心事件

> 2-3 句话说清楚发生了什么。

## 📊 关键数据 / 时间线

- **{YYYY-MM-DD}** — {事件节点 1}
- **{YYYY-MM-DD}** — {事件节点 2}

## 🌊 影响分析

**对行业**

- ...

**对自己 / 团队**

- ...

## 🤔 我的判断

这件事重要吗？为什么？倾向于怎么看？

## 🔮 后续值得关注

接下来应该关注什么信号或进展？
```

#### 模板 D：思想观点

```markdown
## 💭 核心论点

> 作者主要主张什么？用自己的话概括。

## 📚 支撑论据

作者用哪些论据支撑？质量如何？

- **{论据 1}** — {内容}
- **{论据 2}** — ...

## ✨ 启发与应用

读完能用在哪里？改变了什么认知？

## 🔍 反驳与局限

## 💬 一句话总结

> 推荐给同事时，怎么用一句话讲清它的价值？
```

### Step 3：写入本地知识库

**文件路径**：`~/knowledge/YYYY-MM-DD-{slugified-title}.md`

标题 slug 规则：取文章标题，空格替换为 `-`，去除特殊字符，全小写（中文标题保留原文）。

**文件格式**（**禁止使用 YAML frontmatter**，统一用信息卡）：

```markdown
# {文章标题}

> **来源** {原始链接}  
> **日期** {YYYY-MM-DD}  
> **类型** {技术教程 / 原理架构 / 行业资讯 / 思想观点}  
> **标签** {tag1} · {tag2} · {tag3}  
> **关键概念** {concept1} · {concept2} · {concept3}  
> **抓取方式** {chrome-mcp / WebFetch / 用户粘贴}

---

{提炼内容正文}
```

**信息卡书写规则**：
- 用单层 `>` 引用块包裹，每行末尾保留**两个空格**实现换行（Markdown 强制换行语法）
- 字段名加粗，与值之间用一个空格
- `标签`、`关键概念` 多个值之间用 ` · `（中文间隔号，前后各空格）连接，不要用方括号或逗号
- `标签` 反映文章所属领域（最多 4 个）；`关键概念` 提取文章核心术语（3-5 个，便于后续按概念检索）
- `抓取方式` 记录 Step 1 实际成功的层级，便于追溯内容质量
- 字段顺序固定：来源 → 日期 → 类型 → 标签 → 关键概念 → 抓取方式；可省略不适用的字段，但保持此顺序
- 信息卡下方留一空行，再写 `---` 分隔线，再留空行后开始正文

写入完成后告知用户文件路径：

> ✓ 已保存到 `{文件路径}`

---

## 质量标准

- **来源有据**：提炼内容来自原文，不补充原文没有的信息
- **结构精简**：每个要点一句话，不做大段复述
- **类型契合**：选定的模板章节须与文章内容匹配；若多数章节空洞，说明类型选错，重新判断
- **标签准确**：`标签` 反映领域（≤4 个），`关键概念` 反映文章独有术语（3-5 个），二者不要重复
- **emoji 仅章节标题用**：正文不要散布 emoji，保持阅读节奏
- **失败透明**：抓取失败立即说明原因，不伪造内容
