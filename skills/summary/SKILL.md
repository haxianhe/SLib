# summary：链接内容提炼与知识库存档

抓取用户提供的链接，自动识别内容类型，提炼核心观点，写入本地知识库。

---

## 执行流程

### Step 0：信息收集

**先解析输入 → 再自补 → 确认后动手**

#### 解析清单

| 信息项 | 若缺失，如何处理 |
|-------|--------------|
| 链接 URL | **必须追问**，无法继续 |
| 阅读重点/关注角度 | 默认"全面提炼"，在摘要中标注 |
| 保存路径 | 默认 `~/knowledge/`，可在确认时修改 |

#### 理解摘要模板

解析完成后展示摘要，**紧跟 `AskUserQuestion` 确认**：

> 我的理解：
> - 链接：**[URL]**
> - 关注角度：**[全面提炼 / 用户指定角度]**
> - 保存路径：**`~/knowledge/YYYY-MM-DD-{标题}.md`**

```
AskUserQuestion({
  questions: [{
    header: "确认信息",
    question: "以上理解是否正确？",
    multiSelect: false,
    options: [
      { label: "确认，开始抓取", description: "信息无误，立即开始" },
      { label: "需要调整", description: "请在 Other 文本框中说明" }
    ]
  }]
})
```

<HARD-GATE>
未展示摘要并收到用户确认前，不得开始抓取。
</HARD-GATE>

### Step 1：抓取内容

按以下顺序尝试，失败后自动降级，无需询问用户：

#### 层级 1：chrome-mcp（优先）

用浏览器打开页面，利用已有登录态和 JS 渲染能力抓取：

```
mcp__chrome-mcp__get_page_content(url="{URL}")
```

若调用失败（connection refused / ECONNREFUSED / tool not found）：
1. `Bash("pkill -f mcp-chrome-bridge && sleep 2")`
2. 重试一次
3. 仍失败则跳至层级 2

判断失败的条件：返回内容为空、疑似登录页（含"请登录"/"sign in"/"403"等）、内容明显不完整（少于 200 字）。

#### 层级 2：WebFetch（chrome-mcp 失败时）

```
WebFetch({ url: "{URL}", prompt: "返回完整正文内容，保留文章结构" })
```

#### 层级 3：用户粘贴（两层均失败时）

停止抓取，告知用户：

> 自动抓取失败（可能是内容受限或需要特殊登录）。请将文章内容直接粘贴给我。

---

### Step 2：识别类型并提炼

根据内容自动判断文章类型，按对应模板提炼：

**技术文章**（原理解析、工程实践、架构设计等）

```markdown
## 背景与问题
[这篇文章试图解决什么问题，为什么值得写]

## 核心方案
[关键技术思路，3-5 个要点，每点一句话]

## 关键结论
[最值得记住的结论或数据]

## 实践建议
[对自己有用的行动建议或注意事项]
```

**产品 / 案例**（产品分析、公司案例、行业观察等）

```markdown
## 背景
[场景与问题]

## 解决方案
[核心方案或策略]

## 结果与影响
[数据、效果或影响]

## 可借鉴点
[对自己的具体启示]
```

**通用**（无法判断类型时）

```markdown
## 核心观点
[最重要的 3-5 个观点]

## 关键细节
[支撑核心观点的重要信息]

## 个人思考
[延伸思考或值得深入的疑问]
```

---

### Step 3：写入本地知识库

**文件路径**：`~/knowledge/YYYY-MM-DD-{slugified-title}.md`

标题 slug 规则：取文章标题，空格替换为 `-`，去除特殊字符，全小写（中文标题保留原文）。

**文件格式**：

```markdown
---
title: {文章标题}
url: {原始链接}
date: {YYYY-MM-DD}
type: {技术文章 / 产品案例 / 通用}
tags: [{自动识别的 2-4 个标签}]
---

{提炼内容正文}
```

写入完成后告知用户文件路径：

> ✓ 已保存到 `{文件路径}`

---

## 质量标准

- **来源有据**：提炼内容来自原文，不补充原文没有的信息
- **结构精简**：每个要点一句话，不做大段复述
- **标签准确**：tags 反映文章实际内容，不超过 4 个
- **失败透明**：抓取失败立即说明原因，不伪造内容
